# pages/guide/ — the two showcase walkthroughs

> Updated 2026-07-11 · local quick-reference. Track spec: [`../../../../ROADMAP-guide.md`](../../../../ROADMAP-guide.md) + [`../../../../plan/guide.md`](../../../../plan/guide.md). Client notes: [`../../../CLAUDE.md`](../../../CLAUDE.md) §"Guide pages".

Screenshot-driven tours for students and teachers. **Scene copy is data; screenshots are generated, never hand-captured.**

## Files

| File | What it does |
| --- | --- |
| `LearningShowcase.tsx` | `/guide/learning` — student walkthrough (9 scenes, phone-first zigzag) |
| `CreatingShowcase.tsx` | `/guide/creating` — teacher walkthrough (10 scenes, `wide` stacked for desktop editor shots) |
| `learningScenes.ts` | Scene copy data `{en, th}` for the learning tour (count pinned by tests) |
| `creatingScenes.ts` | Scene copy data `{en, th}` for the creating tour |
| `scenes.ts` | Shared scene types/helpers used by both |
| `guideImages.ts` | **Generated** map of scene id → WebP URL (regenerated by `scripts/capture-guide.mjs`) |
| `components/ShowcaseShell.tsx` | Shared page layout both showcases use |
| `components/SceneSection.tsx` | Renders one scene block (image + copy) |
| `components/SceneImage.tsx` | Renders a scene screenshot; picks max-width from the shot's aspect ratio |

## Gotchas
- **Screenshots are generated:** `node scripts/seed-guide-demo.mjs` then `node scripts/capture-guide.mjs` (Playwright → WebP into `public/guide/`). Rerun after UI changes; AI-marked scenes cost real Gemini tokens.
- **Bundle rule:** guide images are `public/guide/*` **URLs**, never `import`ed — keep both showcase routes lazy (guarded by tests). See [`../../../../docs/performance.md`](../../../../docs/performance.md).
- Ids differ per DB — **rerun the seed against production at launch** (`scripts/guide-demo.json`).
